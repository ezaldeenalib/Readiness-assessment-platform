### 1. Button (الأزرار)

**الاستخدام:**
```jsx
import { Button } from '../components/ui';

// Primary button (الزر الأساسي)
<Button variant="primary" size="lg" icon="+" onClick={handleCreate}>
  إضافة جديد
</Button>

// Outline button (زر مخطط)
<Button variant="outline" size="md" onClick={handleCancel}>
  إلغاء
</Button>

// Danger button (زر خطر)
<Button variant="danger" onClick={handleDelete}>
  حذف
</Button>

// Loading state (حالة التحميل)

  جارٍ الحفظ...
```

**الأنواع المتاحة:**

- `ghost` - شفاف (قوائم، تعديل)

## 📄 الصفحات

### 1. 🏦 بنك الأسئلة (`/question-bank`)

**الوصول:** مدراء النظام والوزارة فقط

**المميزات:**
- ✅ عرض جميع الأسئلة مع فلترة متقدمة
- ✅ إنشاء وتعديل الأسئلة
- ✅ 5 أنواع من الأسئلة مدعومة
- ✅ إعداد قواعد التقييم بسهولة
- ✅ توسيع/طي التفاصيل
- ✅ إحصائيات سريعة (عدد الأسئلة، الأنواع، التصنيفات)

**سير العمل:**
1. انقر "إضافة سؤال جديد"
2. اختر نوع السؤال
3. أدخل النص (عربي وإنجليزي)
4. حدد القاعدة والقيمة المرجعية (للأسئلة المرتبطة بالبيانات)
5. أضف الخيارات (للاختيار المتعدد)
6. احفظ

### 2. 🎯 القوالب (`/templates`)

**الوصول:** جميع المستخدمين (قراءة)، المدراء (كتابة)

**المميزات:**
- ✅ عرض شبكي للقوالب
- ✅ عدد الأسئلة وإجمالي النقاط
- ✅ فلترة حسب التصنيف
- ✅ نسخ القوالب
- ✅ تعديل وحذف

**البطاقات تعرض:**
- اسم القالب
- الوصف
- عدد الأسئلة
- إجمالي النقاط
- الإصدار
- الحالة (نشط/معطل)

### 3. ⚙️ بناء القوالب (`/templates/new`)

**الوصول:** المدراء فقط

**المميزات:**
- ✅ معلومات القالب (اسم، وصف، تصنيف)
- ✅ إضافة أسئلة من البنك
- ✅ إعادة ترتيب الأسئلة (↑↓)
- ✅ تنظيم في أقسام
- ✅ تجاوز الوزن لكل سؤال
- ✅ حساب إجمالي النقاط تلقائياً

**سير العمل:**
1. أدخل معلومات القالب
2. انقر "إضافة سؤال"
3. اختر من قائمة الأسئلة
4. رتب الأسئلة
5. حدد الأقسام (اختياري)
6. عدّل الأوزان (اختياري)
7. احفظ القالب

### 4. 📋 تقييمات القوالب (`/template-assessments`)

**الوصول:** جميع المستخدمين

**المميزات:**
- ✅ قائمة جميع التقييمات
- ✅ فلترة (الجهة، القالب، الحالة)

**الوصول:** أصحاب التقييم

**المميزات:**

## 📊 محرك التصحيح التلقائي

### القواعد المدعومة (9 قواعد):

| القاعدة | الرمز | مثال | النتيجة |
|---------|-------|-------|----------|
| أكبر من | > | 650 > 500 | ✅ نجح → 10 نقاط |
| أصغر من | < | 3 < 5 | ✅ نجح → 10 نقاط |
| يساوي | = | "true" = "true" | ✅ نجح → 10 نقاط |
| لا يساوي | ≠ | "A" ≠ "B" | ✅ نجح → 10 نقاط |
| أكبر أو يساوي | ≥ | 45 ≥ 40 | ✅ نجح → 10 نقاط |
| أصغر أو يساوي | ≤ | 2 ≤ 5 | ✅ نجح → 10 نقاط |
| يحتوي | ⊃ | "gov.iq" contains ".gov" | ✅ نجح → 10 نقاط |
| موجود | ∃ | قيمة ≠ null | ✅ نجح → 10 نقاط |
| ضمن نطاق | ⊆ | 50 ≤ 75 ≤ 100 | ✅ نجح → 10 نقاط |

### حساب النتيجة النهائية:

```javascript
// لكل سؤال
if (التقييم ناجح) {
  النقاط المكتسبة = وزن السؤال
} else {
  النقاط المكتسبة = 0
}

// النتيجة الإجمالية
إجمالي المكتسب = مجموع(النقاط المكتسبة)
إجمالي ممكن = مجموع(أوزان جميع الأسئلة)
النسبة المئوية = (إجمالي المكتسب / إجمالي ممكن) × 100

// مستوى المخاطر
if (النسبة ≥ 80) → منخفض ✓
else if (النسبة ≥ 60) → متوسط ⚠
else if (النسبة ≥ 40) → عالي ⚡
else → حرج ✗
```

### المشكلة 3: البيانات لا تُعبأ تلقائياً

**أسباب محتملة:**

**أ) لا توجد بيانات ثابتة:**
```sql
-- تحقق من وجود البيانات
SELECT * FROM static_data_values 
WHERE institution_id = 3 AND is_active = true;
```

**ب) السؤال غير مرتبط بشكل صحيح:**
```sql
-- تحقق من إعدادات السؤال
SELECT id, question_type, linked_static_data_field 
FROM questions 
WHERE id = [question_id];
```

**الحل:** 
1. اذهب إلى `/static-data`
2. اختر الجهة
3. املأ البيانات
4. احفظ
5. أعد إنشاء التقييم

## 📚 أمثلة عملية

### مثال 1: سؤال مرتبط ببيانات ثابتة

```json
{
  "question_text_ar": "هل لدى المنظمة أكثر من 500 موظف؟",
  "question_type": "StaticDataLinked",
  "linked_static_data_field": "total_employees",
  "evaluation_rule": "greater_than",
  "reference_value": "500",
  "weight": 10
}
```

**النتيجة:**
- إذا كان `total_employees = 650` → ✅ 650 > 500 → **10 نقاط**
- إذا كان `total_employees = 420` → ✗ 420 > 500 → **0 نقطة**

### مثال 2: سؤال اختيار متعدد

```json
{
  "question_text_ar": "ما مستوى نضج النسخ الاحتياطي؟",
  "question_type": "MultiChoice",
  "options": {
    "لا يوجد": 0,
    "أساسي": 5,
    "متوسط": 10,
    "متقدم": 15
  },
  "weight": 15
}
```

**النتيجة:**
- اختار "متوسط" → **10 نقاط**
- اختار "متقدم" → **15 نقطة**
- اختار "لا يوجد" → **0 نقطة**

### مثال 3: قالب كامل

```
اسم القالب: تقييم الأمن الأساسي
التصنيف: security
الإصدار: 1.0

الأسئلة:
┌──────────────────────────────────────┬────────┐
│ السؤال                               │ الوزن  │
├──────────────────────────────────────┼────────┤
│ 1. موظفو الأمن السيبراني            │ 20     │
│ 2. شهادة ISO 27001                   │ 20     │
│ 3. عمليات التدقيق الأمني             │ 15     │
│ 4. خطة الاستجابة للحوادث            │ 15     │
│ 5. مستوى النسخ الاحتياطي            │ 15     │

```

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
